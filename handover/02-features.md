# 02. 全機能の仕様

> この章の根拠となるソースコード: `srv/static/index.html`（全体）。
> 仕様書に書かれていない振る舞いは **`index.html` を直接読んで実装してください**。
> 以下の各説明で括弧内に示す行番号は参考目安です。コードの最終形は必ずソースを確認してください。

---

## 1. 商品タイル 1タップ記録

| 項目 | 内容 |
|---|---|
| **目的** | 登録済み商品を 1タップで当日の履歴に即時追加する |
| **導線** | 新規タブ → 商品タイルをタップ |
| **振る舞い** | タップ時に `recordTap(el)`（L391）が呼ばれる。①予算チェック（`checkBudget`）→ 超過済みなら `alert` で拒否（L393–394）、超過予定なら `confirm` ダイアログ（L395）→ OK なら `entries` に `{id, productId, name, emoji, price, qty:1, category, date, time}` を push → `save()` → タイルを `.flash` クラスで 250ms 緑点灯 → Undoトースト表示 → `triggerAlerts` でアラート判定 → `renderTotals` と `renderBudgetBar` を部分更新 |
| **制約** | qty は常に 1。複数個は長押しメニューから。バッジ（使用回数）はタップ直後に DOM を更新するが全再レンダリングはしない（パフォーマンス最適化） |

## 2. 商品の長押しメニュー（ステッパー・×N倍数・任意数量・削除）

| 項目 | 内容 |
|---|---|
| **目的** | 複数個購入・商品編集・商品削除をまとめて行う |
| **導線** | 新規タブ → 商品タイルを 500ms 長押し |
| **振る舞い** | `wireTiles()`（L326）の `pointerdown` で 500ms タイマーをセット。ポインター移動 (3px超) でキャンセル。タイムアウトで `haptic()` → `openProductMenu(pid)`（L419）が呼ばれる。モーダルシートに以下を表示: (1) ステッパー (`−`/`＋` ボタン、デフォルト qty=1) (2) クイック倍数ボタン (×1/×2/×3/×5/×10、L430) (3) 編集ボタン → `openProductEdit(pid)` に遷移 (4) 削除ボタン → `confirm` 後 `products` から除去（履歴は残る）(5) 記録ボタン → `checkBudget(p, price*qty)` → 通過で `entries` に push・`save()`・トースト表示。モーダル外タップで閉じる |
| **制約** | qty は 1〜∞（マイナスボタンは qty=1 で disabled）。ステッパーとクイックボタンは連動（どちらを操作しても `updateQtyDisplay`（L439）で同期）。削除後の履歴エントリは `productId` がぶら下がり状態になるが表示はそのまま保持（名前・絵文字は履歴エントリに複製済み）|

## 3. 商品の新規登録（絵文字ピッカー・カテゴリ選択）

| 項目 | 内容 |
|---|---|
| **目的** | よく買う商品を登録してタイルに追加する |
| **導線** | 新規タブ → 「＋ 追加」タイル → `openProductEdit(null)`（L501）。または長押しメニューの「編集」ボタン |
| **振る舞い** | `openProductEdit(pid)`（L501）を呼ぶ。`pid=null` なら新規、`pid` 指定なら編集。シートに以下のフィールド: (1) 絵文字ピッカー（8列グリッド、約280種類の絵文字）選択中の絵文字は `.sel` で青背景 (2) 名前（テキスト入力、必須）(3) 価格（数値入力、inputmode="numeric"）(4) カテゴリ（`<select>`、`CATEGORIES` から生成）。保存ボタンで `products.push({id:uid(), ...})` または `Object.assign(p, ...)` → `save()` → モーダル閉じ → `render()` |
| **制約** | 名前が空の場合 `alert` → 保存中止。価格は 0 でも保存可能（`+value\|\|0` のため）。既存商品の価格変更は過去の履歴には影響しない（履歴エントリに価格をスナップショット保持） |

## 4. 「メモで記録」モーダル（単発記録、productId なし）

| 項目 | 内容 |
|---|---|
| **目的** | 事前登録していない単発の買い物を素早く記録する |
| **導線** | 新規タブ → 「✏️ メモで記録する（単発の買い物）」ボタン（L353）→ `openQuickEntry(cat)`（L462） |
| **振る舞い** | モーダルシートに以下のフィールド: (1) 名前（必須）(2) カテゴリ（`<select>`、デフォルトは現在のタブが特定カテゴリの場合はそれ）(3) 金額（数値入力、必須・0以下NG）。記録ボタン: `checkBudget` 通過後、`{id:uid(), productId:null, name, emoji:'🛍️', price, qty:1, category, date, time}` を push → `save()` → トースト表示。画面表示時に `#qe-name` を 50ms 後にオートフォーカス |
| **制約** | `productId` は `null`（タイルとの紐付けなし）。絵文字は固定 🛍️。qty は常に 1。予算チェックはタイル記録と同様に行う |

## 5. 予算バー（今月全体＋カテゴリ別ミニ）

| 項目 | 内容 |
|---|---|
| **目的** | 残予算をひと目で把握できるようにする |
| **導線** | 新規タブ・状況タブ上部に常時表示 |
| **振る舞い** | `budgetBarHTML(opts)`（L259）で生成。今月全体バー（大サイズ）: カテゴリタブが「すべて」の場合は全予算合計 vs 全支出、特定カテゴリ選択時はそのカテゴリのみ表示。カテゴリ別ミニバー（2列グリッド）: 「すべて」タブのみ表示、予算が設定されているカテゴリのみ。状況タブ（`opts.fullCategories:true`）では大サイズバーを縦積み。新規タブ（`opts.clickableCategories:true`）ではバーをタップするとカテゴリタブが切り替わる |
| **制約** | `budgets[cat]` が 0 または未設定のカテゴリはバーを表示しない。バー幅は `Math.min(100, pct)%`（100%でクリップ）。進捗バーは CSS の `transition: width .3s` でアニメーション |

## 6. 予算アラート（50/80/100% 一回限り、100% 超でタップ拒否・ダイアログ）

| 項目 | 内容 |
|---|---|
| **目的** | 予算消化の節目をユーザーに通知し、超過時は誤購入を防止する |
| **導線** | `recordTap` または `openProductMenu` の記録ボタン押下時に自動発火 |
| **振る舞い** | `checkBudget(p, addAmount)`（L365）: (1) 記録前支出 `before` が既に `limit` 以上 → `allow:false` で呼び出し元が `alert` 表示・記録拒否 (2) 記録後支出 `after` が `limit` を超える → `allow:'confirm'` で呼び出し元が `confirm` ダイアログを表示。キャンセルで記録中止 (3) 通常 → `allow:true`。`triggerAlerts(cat, before, after, limit)`（L375）: 50/80/100% の閾値ごとに `before < threshold && after >= threshold` かつ `notified[ym+'-'+cat]` に当該閾値未記録の場合のみ `alert` 表示（200ms 遅延）。発火後は `notified[ym+'-'+cat]` に閾値を push して重複発火を防止 |
| **制約** | `notified` のキー形式は `"YYYY-MM-cat"` で値は発火済み閾値の配列（例: `[50, 80]`）。予算編集後は当月の通知フラグを `delete notified[ym+'-'+cat]` でリセット（L782）。メモで記録の場合も同じ検査を行う |

## 7. 予算編集

| 項目 | 内容 |
|---|---|
| **目的** | カテゴリごとの月次予算上限を設定する |
| **導線** | 予算タブ → 各カテゴリカードの「編集」ボタン → `openBudgetEdit(cat)`（L764） |
| **振る舞い** | モーダルシートに上限金額の数値入力フィールドと、クイック選択ボタン（¥3,000 / ¥5,000 / ¥10,000 / ¥20,000 / ¥30,000 / ¥50,000）を表示。保存で `budgets[cat] = +value\|\|0` に更新 → 当月の通知フラグ `notified[ym+'-'+cat]` を削除（再アラートを可能にする）→ `save()` |
| **制約** | 予算を 0 に設定してもエラーにはならない（バーが非表示になるだけ）。クイックボタンはテキスト入力欄の値を上書きするだけ（保存はしない） |

## 8. カテゴリ管理（追加・リネーム・削除）

| 項目 | 内容 |
|---|---|
| **目的** | 支出分類を自分のライフスタイルに合わせて管理する |
| **導線** | 予算タブ → 「⚙️ カテゴリを管理」ボタン → `openCategoryManager()`（L674） |
| **振る舞い** | 既存カテゴリを一覧表示。各行に: (1) リネーム用テキスト入力（`input` イベントで `renameMap` に記録）(2) 何品紐づいているかの件数表示 (3) 削除ボタン（トグル式：押すと「取消」に変わり、再押しでキャンセル）。新規カテゴリ入力欄 → 「追加」ボタンで仮追加（緑バナー表示）。「保存」で一括コミット。削除時のロジック: 削除カテゴリに紐づく商品が存在する場合 `confirm` ダイアログ → OK = 商品も削除、キャンセル = 商品を `newCats[0]`（残存カテゴリ先頭）に移動。リネームは `products`・`entries`・`budgets` の全エントリに波及 |
| **制約** | カテゴリ名の重複チェックあり（`alert` で停止）。名前が空でも停止。`CATEGORIES` 配列が保存の実体（`categories` としてサーバーに保存） |

## 9. 履歴閲覧（カテゴリフィルタ・期間タブ・月/日グループ化・削除）

| 項目 | 内容 |
|---|---|
| **目的** | 過去の支出を振り返り、不要なエントリを削除する |
| **導線** | ボトムナビ「📋 履歴」 |
| **振る舞い** | `renderHistory()`（L537）。上部のカテゴリタブ（`historyTab`）で絞り込み。期間タブ（1ヶ月 / 3ヶ月 / 1年 / 全期間）で時間フィルタ（`time` ミリ秒タイムスタンプで比較）。フィルタ後のエントリを `time` 降順ソート → 月ごとにグループ（`.month-group`）→ その中を日ごとにグループ（`.day`）。各エントリに「削除」ボタンを表示。月グループに月合計、日グループに日合計を表示。ヘッダーに対象期間合計・件数・今月合計を表示 |
| **制約** | 削除は即時反映（`entries` から除去 → `save()` → `renderHistory()` + `renderTotals()`）。Undo 機能なし |

## 10. 集計タブ（月合計・カテゴリ別バー・トップ8）

| 項目 | 内容 |
|---|---|
| **目的** | 今月の支出パターンを視覚的に把握する |
| **導線** | ボトムナビ「📈 集計」 |
| **振る舞い** | `renderStats()`（L591）。今月エントリのみ対象。(1) 月合計カード（合計金額・件数）(2) カテゴリ別横バーグラフ（金額降順、最大値が 100% 基準）(3) 商品別 TOP8 横バーグラフ（`emoji + ' ' + name` をキーに集計、金額降順で上位 8 件）|
| **制約** | 当月エントリが 0 件の場合は「今月の記録はまだありません」を表示 |

## 11. 自動保存（300ms デバウンス + sendBeacon on unload）

| 項目 | 内容 |
|---|---|
| **目的** | 記録を確実にサーバーへ永続化する |
| **導線** | `save()`（L201）は `entries`/`products`/`budgets`/`notified`/`CATEGORIES` を変更する全操作が呼び出す |
| **振る舞い** | `_savePending=true` にセットし 300ms デバウンスタイマーをセット。`flushSave()`（L206）が `PUT /api/state` で JSON を送信。`_saveInflight` フラグでリクエスト多重送信を防止。失敗時は `_savePending=true` にして 500ms 後にリトライ。アンロード時（`beforeunload`）は `_savePending` が true の場合のみ `navigator.sendBeacon('/api/state', JSON)` で同期的に送信 |
| **制約** | `sendBeacon` がないブラウザは `unload` 時の保存を行わない（iOS WKWebView では存在する）。サーバー側のリクエストボディ上限は 16MB |

## 12. PWA（manifest、theme color、apple touch icon）

| 項目 | 内容 |
|---|---|
| **目的** | ホーム画面追加・フルスクリーン表示・ネイティブ風 UX |
| **導線** | ブラウザの「ホーム画面に追加」 |
| **振る舞い** | `manifest.webmanifest` にて `display: standalone`、`theme_color: #10b981`、`background_color: #0f172a` を設定。`<meta name="apple-mobile-web-app-capable" content="yes">` と `apple-mobile-web-app-status-bar-style: black-translucent` でフルスクリーン表示。`apple-touch-icon.png` でホーム画面アイコン |
| **制約** | iOS では Web App Manifest の一部機能（ショートカット等）は未サポート。ネイティブ移植後は不要 |

## 13. Undoトースト

| 項目 | 内容 |
|---|---|
| **目的** | 誤タップによる記録を素早く取り消せる |
| **導線** | 商品タイル 1タップ、長押しメニューからの記録、メモで記録のいずれかの後に自動表示 |
| **振る舞い** | `showToast(msg, onUndo)`（L410）。`#toast` 要素に `.show` クラスを付与してスライドアップ表示。「取り消し」ボタン押下で `onUndo()` コールバックを実行（該当エントリを `entries` から除去 → `save()` → `render()`）。3000ms 後に自動非表示 |
| **制約** | 複数回タップした場合、最後のトーストのタイマーがリセットされる（`clearTimeout(toastTimer)`）。Undo は直前の 1件のみ取り消し可能（複数件の連続 Undo は不可）。履歴タブの「削除」ボタンには Undo なし |
