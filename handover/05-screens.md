# 05. 全画面の要素と並び順

> この章の根拠となるソースコード:
> - `srv/static/index.html`（HTML 構造 L139–151、各 `render*` 関数 L341〜L672、各 `open*` モーダル関数 L419〜L787）
>
> 各画面の DOM 構造の正確な再現は **`index.html` を直接読んで確認してください**。

---

## 共通レイアウト（全画面共通）

`<body>` 直下に縦並びで:

```
<header>             …… タイトルと今日/今月の合計
<div class="tabs">   …… カテゴリタブ（画面によっては display:none）
<main>               …… 画面ごとのコンテンツ
<div class="toast">  …… Undoトースト（fixed, 非表示状態）
<nav class="bottom"> …… ボトムナビゲーション（5タブ）
<div class="modal">  …… モーダルコンテナ（非表示状態）
```

全体は `max-width: 480px` で中央寄せ（PC ブラウザでも携帯幅で表示される）。

## ヘッダー（タイトル＋今日/今月合計）

| 項目 | 値 |
|---|---|
| **画面ID** | `<header>`（L139） |
| **対応DOM** | `index.html` L139–142 |
| **常時表示** | はい |

要素（左→右）:
1. 「今日」ラベル + `<b id="today-total">¥0</b>`（更新は `renderTotals()` L250）
2. 「今月」ラベル + `<b id="month-total">¥0</b>`（右寄せ）

注意: アプリ名「サクサク家計簿」は `<title>` のみで、ヘッダーには表示されていない。**ヘッダーに表示するのは「合計金額」だけ**。

## カテゴリタブ（`<div class="tabs" id="tabs">`）

| 項目 | 値 |
|---|---|
| **対応DOM** | `index.html` L144（中身は `renderTabs()` または `renderHistory()` が生成） |
| **表示画面** | 新規タブ・履歴タブ |

## ボトムナビ（5タブ固定）

| 項目 | 値 |
|---|---|

## 1. 状況タブ

| 項目 | 値 |
|---|---|
| **画面ID** | `status` |
| **対応関数** | `renderStatus()`（`index.html` L341） |

要素（上→下）:
1. **ヘッダー**（共通）
2. **カテゴリタブ非表示**
3. **メイン**: 「すべて」状態固定の `budgetBarHTML({fullCategories: true})` 出力 = 大サイズの予算バー縦積み:
   - 今月全体バー
   - 各カテゴリの予算バー（予算が設定されているもののみ、`CATEGORIES` 順）

タップ要素なし（純粋な閲覧画面）。

---

## 2. 新規タブ（デフォルト）

| 項目 | 値 |
|---|---|
| **画面ID** | `new` |
| **対応関数** | `renderNew()`（`index.html` L351） |

要素（上→下）:
1. **ヘッダー**（共通）
2. **カテゴリタブ非表示**（タブ切替は予算バーをタップでも可能）
3. **メイン**:
   1. 予算バースロット（`<div id="budget-bar-slot">`）: `budgetBarHTML({clickableCategories: true})` 出力 = 今月全体バー + カテゴリ別ミニバー（2列）。タップでカテゴリタブが切り替わる。
   2. 「✏️ メモで記録する（単発の買い物）」ボタン（青紫グラデーション、`#quick-entry-btn`）
   3. 商品タイルグリッド（`tilesHTML()`、L311）: 3列、使用頻度降順、最後に「＋ 追加」タイル

タイルの挙動:
- タップ → `recordTap`（1件記録）
- 長押し（500ms）→ `openProductMenu`
- 「＋ 追加」タイル → `openProductEdit(null)`

---

## 3. 予算タブ

| 項目 | 値 |
|---|---|
| **画面ID** | `budget` |
| **対応関数** | `renderBudget()`（`index.html` L616） |

要素（上→下）:
1. **ヘッダー**（共通）
2. **カテゴリタブ非表示**
3. **メイン**:
   1. **今月サマリーカード**（`.summary`、青紫グラデーション）:
      - 「YYYY-MM 今月の予算」
      - 大きい数字: `¥支出 / ¥予算合計`
      - 進捗バー（白）
      - 経過日数: `N日目 / N日`, `経過率 XX%`
      - サブメッセージ: ペース判定（速い/順調/計画通り）+ 「1日あたり ¥XXX まで使えます」
   2. **カテゴリカード**（`.bcard`、`CATEGORIES` の順）各カード:
      - カテゴリ名 + `¥支出 / ¥予算` + 「編集」ボタン
      - 進捗バー（色は閾値依存）
      - メタ: `使用率 XX%`, `残り ¥XXX`
      - アラートメッセージ（80%超など）
   3. **「⚙️ カテゴリを管理」ボタン**（`#manage-cats-btn`、グレー）

「編集」ボタン → `openBudgetEdit(cat)`、「カテゴリを管理」ボタン → `openCategoryManager()`。

---

## 4. 履歴タブ

| 項目 | 値 |
|---|---|
| **画面ID** | `history` |
| **対応関数** | `renderHistory()`（`index.html` L537） |

要素（上→下）:
1. **ヘッダー**（共通）
2. **カテゴリタブ表示**（`historyTab` で管理）
3. **メイン**:
   1. **期間タブ**（`.period-tabs`、丸ピル形式）: 1ヶ月 / 3ヶ月 / 1年 / 全期間
   2. **集計ヘッダーカード**（`.month-group` 風）:
      - 左: カテゴリ名 + 期間ラベル / 合計 / 件数
      - 右: 「今月」 / 今月合計
   3. **空状態時**: `.empty` で「まだ記録がありません」など
   4. **月グループ**（`.month-group`、新しい月から順）各月:
      - 月ヘッダー（`.mh`）: `YYYY-MM` + 月合計
      - **日グループ**（`.day`、新しい日から順）各日:
        - 日ヘッダー（`<h3>`）: `YYYY-MM-DD` + 日合計
        - **エントリ行**（`.entry`、その日の中で時刻降順）:
          - 絵文字 / 名前 (×qty があれば付加) / `HH:MM · カテゴリ`
          - 金額 + 「削除」ボタン

「削除」ボタンは即時実行（Undo なし）。

## 5. 集計タブ

| 項目 | 値 |
|---|---|
| **画面ID** | `stats` |
| **対応関数** | `renderStats()`（`index.html` L591） |

要素（上→下）:
1. **ヘッダー**（共通）
2. **カテゴリタブ非表示**
3. **メイン**:
   1. **月合計カード**（`.stat-card`）: 「今月（YYYY-MM）合計」+ 大きい金額 + 件数
   2. **カテゴリ別カード**: 「カテゴリ別」見出し + 横バーリスト（金額降順）。各行: ラベル / バー / 金額

空状態時（当月エントリ 0）: 「今月の記録はまだありません」表示で他は出さない。

## 6. 商品登録モーダル

| 項目 | 値 |
|---|---|
| **対応関数** | `openProductEdit(pid)`（`index.html` L501） |
| **タイトル** | 編集時「商品を編集」/ 新規時「商品を追加」 |

要素（上→下）:
1. タイトル
2. **アイコン**（絵文字ピッカー、8列グリッド、`max-height: 240px` でスクロール）
3. **名前**（テキスト入力、必須）
4. **価格（円）**（数値入力、`inputmode="numeric"`）
5. **カテゴリ**（`<select>`）
6. **アクション行**: 「キャンセル」（グレー）／「保存」（青）

モーダル外タップで閉じる。名前空欄で保存しようとすると `alert`。

## 7. 商品長押しメニュー

| 項目 | 値 |
|---|---|
| **対応関数** | `openProductMenu(pid)`（`index.html` L419） |
| **タイトル** | 「{絵文字} {名前} ¥{価格}」 |

要素（上→下）:
1. タイトル
2. **ステッパー**（`.qty-stepper`、高さ 120px の大型ボタン）:
   - 左: `−` ボタン（薄グレー、qty=1 で disabled）
   - 中央: `×{qty}` / `¥価格 × qty` / 緑色の合計金額
   - 右: `＋` ボタン
3. **クイック倍数ボタン**（`.qty-quick`、横並び）: ×1 / ×2 / ×3 / ×5 / ×10
4. **アクション行**: 「編集」（グレー）／「削除」（赤）／「記録」（青）

ステッパーとクイックボタンは双方向同期。「記録」で予算チェック → 記録 → トースト。「削除」で `confirm`「削除しますか？（履歴は残ります）」。

## 9. カテゴリ管理モーダル

| 項目 | 値 |
|---|---|
| **対応関数** | `openCategoryManager()`（`index.html` L674） |
| **タイトル** | 「カテゴリを管理」 |

要素（上→下）:
1. タイトル
2. **既存カテゴリリスト**（`max-height: 50vh` スクロール）各行:
   - リネーム用テキスト入力
   - `{N}品` 件数ラベル
   - 「削除」赤ボタン（トグル式：押下で「取消」表示・薄表示）
3. **新規追加セクション**:
   - 「新しいカテゴリを追加」ラベル
   - 入力欄 + 「追加」青ボタン（同行）
4. **アクション行**: 「キャンセル」／「保存」

仮追加カテゴリは緑バナーで `＋ {名前}（保存で追加）` と表示される。「保存」を押すまでは何もコミットされない。

| 項目 | 値 |

| **対応関数** | `openBudgetEdit(cat)`（`index.html` L764） |
| **タイトル** | 「{カテゴリ名}の予算を設定」 |

要素（上→下）:
1. タイトル

## 11. 確認ダイアログ（予算超過時など）

ネイティブの `confirm()` / `alert()` を使用（カスタムダイアログではない）。iOS 実装では `UIAlertController` (UIKit) または `.alert(...)` (SwiftUI) で代替。

主なダイアログ:
- **予算超過の事前確認**: `confirm('予算を¥XXX超過します。記録しますか？')`
- **予算超過のタップ拒否**: `alert('⛔ {cat}の予算上限¥XXXに達しています。\n現在: ¥XXX')`
- **アラート閾値到達**: `alert('🔔 {cat}が予算の{n}%に到達 ...')` または `alert('⚠️ {cat}が予算上限に達しました！')`
- **商品削除**: `confirm('削除しますか？\n（履歴は残ります）')`
- **カテゴリ削除時の商品扱い**: `confirm('削除するカテゴリに{N}件の商品が紐づいています。\n商品も一緒に削除しますか？\n\nキャンセル=商品は「{fallback}」に移動')`

## 12. Undo トースト

| 項目 | 値 |
|---|---|
| **対応DOM** | `<div class="toast" id="toast">`（`index.html` L146） |
| **対応関数** | `showToast(msg, onUndo)`（`index.html` L410） |

要素（左→右）:
1. メッセージ（`.msg`、1行省略表示）
2. 「取り消し」ボタン（白半透明背景）
