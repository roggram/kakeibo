# 07. ユニットテストで検証すべき項目

> この章の根拠となるソースコード: `srv/static/index.html` のロジック関数群。
> 各テスト項目で検証すべき関数の挙動は **`index.html` を直接読んで確認してください**。
>
> iOS では XCTest（ユニット）と XCUITest（UI）に分けて実装することを想定しています。

---

## A. 集計・計算ロジック（純粋関数、XCTest）

### A-1. 当月支出（カテゴリ別 / 全体）

対応関数: `categorySpent(cat, ym)` / `renderTotals()`

| ケース | 期待 |
|---|---|
| 当月内のエントリのみ合計される | `price * qty` の総和 |
| 前月・来月のエントリは含まれない | 除外 |
| カテゴリ指定で他カテゴリは除外 | カテゴリ一致のみ |
| `entries` 空 | `0` |
| `qty` が 2 以上のエントリ | `price * qty` として加算される |

### A-2. 予算超過判定（`checkBudget`）

| ケース | 期待 `allow` |
|---|---|
| `budgets[cat]` 未設定（または 0）| `true`（無制限）|
| 記録前 `before < limit` かつ `after <= limit` | `true` |
| `before < limit` かつ `after > limit` | `'confirm'` |
| `before >= limit` | `false` |

メモで記録（`productId: null`、`pseudo = {category, price}`）でも同じ結果を返すこと。

### A-3. 50/80/100% 閾値発火条件（`triggerAlerts`）

| ケース | 期待 |
|---|---|
| `before < 0.5*limit` かつ `after >= 0.5*limit` で `notified` 未記録 | 50% アラート発火 + `notified[ym+'-'+cat]` に `50` push |
| 80%, 100% も同様 | それぞれ 1回ずつ発火 |
| 既に同月同閾値が `notified` にある | 再発火しない |
| 一度のタップで 50%→80% を一気に跨ぐ | 両方発火する |
| 月が変わると過去月の `notified` キーは影響しない | 新月の閾値は再発火可能 |

### A-4. 使用頻度ソート（`usageCount`）

| ケース | 期待 |
|---|---|
| 各 `productId` の `entries` 出現回数を返す | 件数 |
| `productId: null` のエントリは数えない | 任意 product にカウントしない |
| 使用頻度降順でソートされる | 多い順 |
| 同数の商品は元の順序を保つ | sort 安定性 |

### A-5. 集計タブのトップ8（`renderStats`）

| ケース | 期待 |
|---|---|
| 集計キーは `emoji + ' ' + name` | 文字列キー |
| 当月内のみ集計 | 期間外は除外 |
| 金額降順で上位 8 件 | `slice(0, 8)` |
| 9件未満なら全件を返す | OK |
| 商品名・絵文字を編集した過去エントリは別キー扱い | 仕様 |

### A-6. ペース分析（`renderBudget` 内ロジック）

| ケース | 期待 |
|---|---|
| `dailyAllow = round(remain / (daysInMonth - day + 1))` | 残額÷残日数 |
| `projection = round(totalSpent / day * daysInMonth)` | 当月予測 |
| `totalPct > dayPct + 15` | ペース速い判定 |
| `totalPct < dayPct - 15` | 順調判定 |
| `day == 1` で動作する（projection の除算が落ちない）| OK（`day > 0` ガードあり）|

## B. 状態遷移（XCTest）

### B-1. notified フラグの月限定性

- 同月・同カテゴリで同じ閾値は二度発火しない
- 月が変わった瞬間（`ym` 文字列が変わる）に新キーに切り替わり、過去月の通知履歴は影響しない
- 予算編集後は `notified[ym+'-'+cat]` が削除され、当月の再アラートが可能になる

### B-2. カテゴリ削除時の商品移行/削除

`openCategoryManager` 保存ロジック:

| ケース | 期待 |
|---|---|
| カテゴリ A を削除、A 紐付き商品なし | 商品配列無変化 |
| カテゴリ A を削除、A 紐付き商品あり、ユーザーが「商品も削除」を選択 | `products` から該当除去、履歴は残る |
| カテゴリ A を削除、A 紐付き商品あり、ユーザーが「移動」を選択 | 商品の `category` が `newCats[0]` に変更 |
| カテゴリ A をリネーム | `products`/`entries`/`budgets` 全てに波及 |
| 削除カテゴリの `budgets` キー | `delete budgets[cat]` で削除 |
| 重複名で保存 | バリデーションエラー（`alert`）で停止 |

### B-3. 商品削除と履歴の関係

- 商品を削除しても履歴の該当エントリは保持される（name/emoji/price はスナップショット済み）
- 削除後、その `productId` の `usageCount` は引き続き返るが UI には現れない

### B-4. Undo

- 1タップ記録直後の Undo で該当 entry のみ除去（直前の操作 1件のみ）
- メモで記録の Undo も同様
- 長押し記録の Undo は `entries.pop()` で末尾のみ削除（操作直後の前提）
- 3000ms 経過後はトースト非表示でも Undo は機能しない（DOM が消える）

## C. 永続化（XCTest + 模擬サーバー）

### C-1. ステートのラウンドトリップ

- `JSON.stringify({products, entries, budgets, notified, categories: CATEGORIES})` → サーバー → `GET` → 元のオブジェクトと深く一致する
- フィールド順は保証しないが、データ内容が同一であること

### C-2. 初回 null ハンドリング

- `GET /api/state` が `"null"` を返す場合、デフォルト値（8件の商品・初期予算・既定 5カテゴリ）が使われる
- 最初の `save()` でサーバーにデータが書き込まれる

### C-3. デバウンス挙動（擬似）

- 300ms 以内の連続 `save()` 呼び出しは 1回の `flushSave()` にまとめられる
- `_saveInflight` 中の `save()` は `_savePending = true` を残し、完了後に再フラッシュされる
- `flushSave` 失敗時は `_savePending = true` のまま 500ms 後に再試行

### C-4. アンロード保存

- `beforeunload` 時に `_savePending == true` なら `sendBeacon` が呼ばれる
- `_savePending == false` なら呼ばれない

## D. UI / インタラクション（XCUITest）

### D-1. 1タップで履歴に 1件

1. 新規タブで適当な商品タイルをタップ
2. 履歴タブを開く
3. 最上位に絵文字・名前・本日の日付で 1件存在する
4. 金額が `price` と一致する、qty=1 表示なし（×1 は表示しない）

### D-2. 長押しメニューの開閉

1. 商品タイルを 500ms 長押し
2. シートが下からスライドアップする
3. デフォルト qty が 1
4. ×3 ボタンをタップ → ステッパー表示が `×3` に同期
5. ＋ボタンを 2回タップ → `×5` になる
6. ステッパーとクイックボタンの状態が連動する

### D-3. ×N倍数ボタン記録

1. 長押しメニューで ×5 を選択 → 「記録」
2. 履歴に 1件追加され `名前 ×5` と金額 `price * 5` が表示される

### D-4. Undo トースト

1. タイルをタップ
2. 緑トーストが画面下に現れる
3. 「取り消し」をタップ
4. 履歴から該当 entry が消える
5. トーストが消える
6. 3秒待っても再表示はされない

### D-5. 予算超過ダイアログのキャンセル/許可

事前準備: あるカテゴリの予算を ¥1,000 に設定、現在の支出を ¥900 とする。¥500 の商品をタップする。

| 操作 | 期待 |
|---|---|
| `confirm` が表示される | テキストに「予算を¥400超過します」を含む |
| キャンセル | entry 追加されない、トースト出ない |
| OK | entry 追加、トースト表示、100% アラートが発火 |

事前準備 2: 予算 ¥1,000、現在支出 ¥1,000、¥500 商品をタップ。

| 操作 | 期待 |
|---|---|
| `alert` が表示される（`⛔ ...`）| OK のみ |
| entry 追加されない | 履歴に変化なし |

### D-6. カテゴリ管理

1. 予算タブ → カテゴリ管理を開く
2. 「食品」をリネーム入力で「食料品」に書き換え
3. 「保存」
4. 商品の `.category` がすべて「食料品」に、`budgets.食料品` に移行している
5. 履歴の `category` も全て「食料品」になっている

### D-7. 期間タブ・カテゴリタブ

1. 履歴タブで「1ヶ月」「3ヶ月」「1年」「全期間」を切り替え、件数と合計が変動する
2. カテゴリタブで絞り込み、当該カテゴリのエントリのみ表示

### D-8. PWA としての起動（参考。iOS ネイティブでは不要）

- iOS Safari でホーム画面に追加できる
- ステータスバーが半透明黒で全画面表示される
