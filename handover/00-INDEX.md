# サクサク家計簿 — iOS 移植引き継ぎ資料

このドキュメントは Web 版サクサク家計簿を iOS ネイティブアプリに移植するための全仕様書です。

## この資料の使い方（Claude Code 向け）

**重要：仕様書だけを読んで実装しないでください。**

仕様書に明示されていない振る舞い（並び順、エッジケース、暗黙のルール、アニメーション、エラーメッセージの文言、色の遷移閾値など）を実装するときは、必ず **移植元のソースコードを読みに行ってから実装してください**。仕様書を「再解釈」するとささいなディテールがずれます。

ソースコードは VM 内にあります（各ファイルパスは「移植元のソースコード」セクション参照）。

### 進め方

ロジック先行・1画面ずつ進める方針です。
画面を実装する前に、**毎回ユーザーに「今からこの画面を作ります。この画面のスクリーンショットを送ってください」と言うこと。**
スクリーンショットを確認してからコードを書いてください。画面ごとにユーザーに確認を取りながら進めてください。

## スクリーンショット

スクリーンショットは `screens/` ディレクトリに VM 内に保存されています。ユーザーから「この画面のスクリーンショットを送ってください」と言われたら、その画面を実装し始める前にユーザーにスクリーンショットを要求してください。

複製・表示が必要になったら:
```
ls /home/exedev/kakeibo-handover/screens/
```
で一覧を見られます。

## 使用技術スタック (現状)

- フロントエンド: HTML + CSS + Vanilla JS (フレームワーク未使用、主要コードは `index.html` 1 枚)
- バックエンド: Go 1.26 + SQLite (`modernc.org/sqlite`)
- データ保存: サーバー側 SQLiteの `app_state` テーブルに JSON オブジェクト 1行 (シングルユーザー前提)
- 認証: なし

## ユーザー認証について

現在はシングルユーザー前提のため認証はありません。iOS アプリも同様にユーザー認証なし、デバイスローカルで動作させてください（iCloud同期は任意、推奨しますがユーザー認証は不要）。
